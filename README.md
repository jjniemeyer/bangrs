# bangrs

A minimal desktop music player built in Rust. A learning project — the goal is to explore [Slint](https://slint.dev/) for desktop UI, [cpal](https://github.com/RustAudio/cpal) for audio output, and [symphonia](https://github.com/pdeljanov/Symphonia) for decoding, under a GPL-licensed workspace.


## Status

- Scans a folder for `mp3` / `flac` / `wav` / `ogg` files and reads tags via lofty.
- Plays through the default audio device (no resampler yet — see caveats).
- Slint UI with Pick Library / Play / Pause / Stop and a track list.
- 29 unit + integration tests green; 1 ignored cpal hardware smoke test.
- Workspace-pinned dependencies, per-crate `thiserror` error enums, no `unwrap()` in library code.

## Workspace layout

Four crates under `crates/`, each depending only on what's below it:

| Crate | Role |
|---|---|
| `bangrs-core` | Domain types (`Library`, `Track`, `Player` state machine, `ViewModel` reducer, `Event`). No I/O. |
| `bangrs-scan` | Filesystem scanning — `walkdir` + `lofty` probe. `Scanner` trait with `FilesystemScanner` and `FakeScanner`. |
| `bangrs-audio` | Playback — `AudioEngine` trait (`CpalEngine` / `FakeEngine`), `Output` trait, `Command` enum. |
| `bangrs-ui` | Slint GUI binary. `PlayerHandle` wraps the command channel; `main.rs` wires callbacks + event loop. |

## Requirements

- Rust stable (edition 2024; tested on a recent `1.9x`-series stable).
- Linux / macOS / Windows desktop with a working default output device.
- Linux: ALSA + pulse/pipewire dev libs for cpal, plus Wayland/X11 deps for Slint. On Arch this is roughly:
  ```
  pacman -S alsa-lib libxkbcommon fontconfig
  ```

## Build & run

```bash
cargo run -p bangrs-ui              # debug build
cargo run --release -p bangrs-ui    # release build (takes a while; Slint's dep tree is large)
```

1. Click **Pick Library**, choose a folder containing audio files.
2. The track list populates.
3. Double-click a track to play. Pause / Stop work as expected.

## Tests

```bash
cargo test --workspace --no-fail-fast                       # everything
cargo test -p bangrs-core                                   # one crate
cargo test -p bangrs-audio --test cpal_smoke -- --ignored   # hardware smoke (needs real output device)
```

Scan fixtures (`crates/bangrs-scan/tests/fixtures/`) are generated by `gen-fixtures.sh` and committed alongside the script. Regenerate with `sox`, `ffmpeg`, `id3v2` installed.

## Caveats

- **Limited resampling.** The engine negotiates a matching cpal stream config for each track's sample rate (Path A). On Linux/PipeWire (the primary developer setup), this works for any rate the kernel exposes. On JACK with a fixed session rate, tracks at a different rate emit `TrackFailed` with a "no cpal config supports rate {N} Hz; available rates: ...; track skipped" reason — pure-resampling support (Path B, via `rubato`) is reserved for a later phase.
- **Seek is a no-op.** `PlayerHandle::seek` sends a command; the engine ignores it.
- **No auto-advance.** When a track ends, playback stops — the UI doesn't queue up the next row.
- **Position isn't rendered in the UI yet.** The engine emits `PositionUpdate` events and the `ViewModel` tracks them; just no slider widget binding.

These are deliberate — the project follows a red/green TDD rhythm, and each slice ships only what its tests cover.

## Architecture notes

- **Audio thread boundary**: UI → engine via `mpsc::SyncSender<Command>`; engine → UI via `crossbeam_channel::Sender<Event>` plus an `Arc<AtomicU64>` for the playback position. The Slint event loop drains events on a 50 ms timer and rebinds the `ViewModel`.
- **`Player` is an enum**, not typestate generics. Transitions are `fn(self) -> Result<Player, CoreError>` — invalid edges return `CoreError::InvalidTransition`.
- **`ViewModel` is a pure reducer**: `fn apply(self, &Event) -> Self`, `fn set_library(self, Arc<Library>) -> Self`. UI binding is one-way.
- **`cpal::Stream` is `!Send`**, so `CpalOutput::new()` returns `(CpalOutput, Stream)` and the engine keeps the `Stream` alive on the audio thread.
